### Vulnerability Details
The function validate_past_idat in spng.c uses a local pointer variable `data` without initializing it before dereferencing. In the tIME chunk handling, the code reads from `data`:

    static int validate_past_idat(spng_ctx *ctx)
    {
        ...
        unsigned char *data;
        ...
        else if(!memcmp(chunk.type, type_time, 4))
        {
           if(ctx->file_time) return SPNG_EDUP_TIME;
    
           if(chunk.length != 7) return SPNG_ECHUNK_SIZE;
    
            struct spng_time time;
    
            time.year = read_u16(data);
            memcpy(&time.month, data + 2, 1);
            memcpy(&time.day, data + 3, 1);
            memcpy(&time.hour, data + 4, 1);
            memcpy(&time.minute, data + 5, 1);
            memcpy(&time.second, data + 6, 1);
            ...

The local pointer `data` is never assigned a value in this function. Although other codepaths use `ctx->data` (populated by read_chunk_bytes/read_data) as the chunk data buffer, this function erroneously uses the uninitialized local `data`, leading to a null-dereference or arbitrary memory read. This is reachable when parsing an IDAT followed by a tIME chunk (length 7) and causes a crash (denial-of-service) and undefined behavior.

Notes / Suggested Fix:
- The immediate fix is to use the correct chunk data pointer when reading the tIME chunk (e.g., replace `data` with `ctx->data` or assign `data = ctx->data;` after reading chunk bytes), ensuring the code reads from the buffer populated by read_chunk_bytes/read_data.

- This bug causes a crash (DoS) when parsing PNGs that exercise this path and should be fixed to prevent undefined behavior.
