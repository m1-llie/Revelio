### Vulnerability Details
The function validate_past_idat() in spng.c uses an uninitialized pointer 'data' when parsing ancillary chunks (tIME and eXIf) that appear after the last IDAT. Specifically, 'data' is declared but never assigned before being dereferenced. This results in undefined behavior and can cause out-of-bounds memory reads or crashes depending on the incidental stack contents.
